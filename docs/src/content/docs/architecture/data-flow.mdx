### Async: faithfulness evaluation

After the response is returned, a background task scores answer grounding:

```
Agent API background task
  → LLMObs.task("faithfulness-eval")
  → gpt-4.1-nano: score 0–1 based on source fidelity
  → LLMObs.submit_evaluation("faithfulness", score)
  → Appears under Evaluations tab in Datadog LLM Obs
```

## Data ingestion pipeline

Raw government data flows from external APIs through transformation into a searchable vector index.

<Mermaid code={`

    subgraph src["Government Sources"]

    end

    subgraph dag["Airflow DAG Pattern (per source)"]
        direction LR
        F["1 · fetch_data\\npaginate external API\\nXCom push records"] --> G["2 · store_raw_parquet\\nDataFrame → Parquet\\nupload to Blob Storage"] --> H["3 · index_to_search\\nchunk · embed · upsert\\nAzure AI Search"]
    end

    subgraph az["Azure"]
        I[("Blob Storage\\nraw-data/&lt;domain&gt;/YYYYMMDD.parquet")]
        J[("AI Search Index\\n615k+ docs\\nhybrid BM25 + 1536-dim vector")]
    end

    A & B & C & D & E --> F
    G --> I
    H --> J
`} />

Each record goes through the same three-task Airflow pattern:

1. **`fetch_data`** — paginates the external API and XCom-pushes a list of record dicts
2. **`store_raw_parquet`** — converts to Pandas, serializes to Parquet, uploads to `raw-data/<domain>/filename_YYYYMMDD.parquet` via `dd_upload_blob()` (emits `azure.blob.upload` APM span)
3. **`index_to_search`** — chunks (character window or tiktoken 512-token/64-overlap), embeds with `text-embedding-3-small`, and upserts in 100-doc batches

## Session memory

Redis stores two types of data for the Agent API:

| Key pattern | TTL | Content |
|---|---|---|
| `infra-advisor:session:{uuid}:memory` | 86 400 s | JSON list of `HumanMessage` / `AIMessage` exchanges |
| `infra-advisor:session:{uuid}:model` | 86 400 s | String: Azure OpenAI deployment name |
| `infra-advisor:suggestions:pool` | none | Redis List: up to 80 infrastructure-focused suggestions |

The suggestion pool is maintained by a background asyncio task (`_pool_maintenance_loop`) that:

- Seeds the pool on startup if empty
- Refills to 80 items every 30 minutes when pool drops below 20
- Uses 4 rotating batch prompts (Engineering / Construction / Operations / Management focus)

Page loads draw 4 random items from the pool in ~1 ms with no LLM wait.
